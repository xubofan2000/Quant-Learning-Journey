# Day 1 - 向量点乘与组合收益率

> Week 3 主题：从理论到代码 — 线性代数与资产组合映射
> 今天是第一天：把"股票权重×收益率"这个最基础的金融操作，翻译成线性代数语言。

## 🎯 核心认知

**组合收益率 = 权重向量 · 收益向量 = 标量**

```
w = [w1, w2, w3]     ← 1×N 行向量（你的仓位配置）
r = [r1, r2, r3]^T   ← N×1 列向量（市场当天给你的随机信号）
R_p = w · r           ← 标量（你当天的盈亏）
```

这就是线性代数在量化领域的**第一个落地场景**。

## 🧮 三种等价写法

```python
# 方法 1: for 循环（能跑但慢）
total = 0
for i in range(N):
    total += w[i] * r[i]

# 方法 2: np.dot
total = np.dot(w, r)

# 方法 3: @ 运算符（推荐，Python 3.5+）
total = w @ r
```

三种方法结果完全一致，但性能差距巨大。

## ⏱️ 性能实测

计算全量 ~1700 个交易日的组合收益：

| 方法 | 耗时 | 加速比 |
|------|------|--------|
| 双重 for 循环 | ~ms 级 | 1x |
| 单层 for + np.dot | 更快 | ~数倍 |
| **全矩阵 R @ w** | ~μs 级 | **100x+** |

**原因**：NumPy 底层调用 C/Fortran 的 BLAS 库（如 OpenBLAS/MKL），向量化操作直接在内存中批量计算，消除了 Python 解释器的循环开销。

## 📐 维度 Cheat Sheet

| 对象 | 含义 | NumPy shape |
|------|------|-------------|
| `w` (权重) | 投资者的"旋钮" | `(N,)` |
| `r_t` (单日收益) | 市场随机信号 | `(N,)` |
| `R` (收益率矩阵) | 历史信号堆叠 | `(T, N)` |

| 运算 | 代码 | 结果 shape |
|------|------|-----------|
| 单日组合收益 | `w @ r_t` | 标量 `()` |
| 全量组合收益 | `R @ w` | `(T,)` |
| 组合方差（Day 4） | `w @ Σ @ w` | 标量 `()` |

## ⚠️ 核心铁律：两种收益率各管一个维度

> **对数收益率在时间上有可加性，但不能用于横截面的组合加权平均；简单收益率则反之，可以加权平均组合收益率，在时间序列上需要累乘。**

| | 时间维度（纵向聚合） | 横截面维度（横向加权） |
|---|---|---|
| **对数收益率** | ✅ 直接 `sum()` | ❌ 不能 `w @ r_log` |
| **简单收益率** | ❌ 必须 `cumprod(1+r)` | ✅ 直接 `w @ r_simple` |

**数学根因**：`ln()` 是凹函数。由 Jensen 不等式：

```
ln(w1*x1 + w2*x2) >= w1*ln(x1) + w2*ln(x2)
```

等号仅在所有 xi 相等时成立。所以对数收益率的加权平均 ≠ 组合的真实对数收益率。

**实操准则**：
- 算"组合今天赚了多少"（横截面加权）→ 用**简单收益率**：`R_p = w @ r`
- 算"这只股票全年赚了多少"（时间累计）→ 用**对数收益率**：`exp(sum(log_r)) - 1`
- 算"组合净值曲线"→ 先用简单收益率加权得到组合日收益，再 `cumprod(1+R_p)`

## 🔍 关键发现

1. **权重 w 是投资者唯一能控制的变量** — 收益率 r 是市场给的，你只能调 w
2. **不同权重方案 → 完全不同的风险收益特征**（年化收益、波动率、夏普比全变）
3. **等权组合的点乘 = 算术平均** — `w_equal @ r == r.mean()`，这是一个有用的 sanity check
4. **净值曲线用简单收益率累乘** — `NAV = cumprod(1 + simple_returns)`（不是 log return 的 cumsum）
