# Day 2 - 逐元素乘法(`*`) vs 矩阵乘法(`@`)

> Week 3 主题：从理论到代码 — 线性代数与资产组合映射
> 今天是第二天：理解 `*` 和 `@` 两种乘法的本质区别，以及 Pandas 底层就是 NumPy 的真相。

## 🎯 核心认知

**`*` 是展开，`@` 是压缩**

```
* 逐元素乘法 (Hadamard 积):  保留每个位置的独立贡献
  [0.5, 0.3, 0.2] * [0.01, -0.02, 0.03] = [0.005, -0.006, 0.006]
  shape 不变: (N,) → (N,)

@ 矩阵乘法 (内积/点积):   把所有贡献加总为一个数字
  [0.5, 0.3, 0.2] @ [0.01, -0.02, 0.03] = 0.005
  内维坍缩: (N,) → 标量
```

金融直觉：
- `*` → "每只股票各自贡献了多少收益"（分项明细）
- `@` → "把所有贡献加总为组合总收益"（汇总数字）

## 🧮 等价关系

```python
# 这三种写法结果完全一致：
w @ r                    # 一步到位（推荐）
np.sum(w * r)            # 两步拆解
np.dot(w, r)             # 等价于 @
```

**但性能不同**：`@` 底层调用 BLAS 的 dot 产品，比先 `*` 再 `sum()` 更快更高效。

## 🌊 Broadcasting 的展开机制

当 `*` 作用于不同 shape 的数组时，NumPy 会自动 broadcasting：

```
R (50, 50)  *  w (50,)  →  (50, 50)

w 被自动扩展为 (1, 50)，再逐行复制为 (50, 50)
每一行 = 某天每只股票的收益贡献值
```

关键对照：

| 运算 | 结果 shape | 维度变化 | 金融含义 |
|------|-----------|---------|---------|
| `R * w` | (T, N) | 不变 | 每天每只股票的贡献明细 |
| `R @ w` | (T,) | 内维坍缩 | 每天的组合总收益 |
| `R @ R.T` | (T, T) | 双重坍缩 | 日期间收益关联结构 |

## ⏱️ 性能实测（50天×50只）

| 方法 | 特点 | 相对速度 |
|------|------|---------|
| 双重 for 循环 | Python 解释器逐元素 | 1x（基准） |
| 单层 for + sum(w*r) | 半向量化 | 数倍 |
| `(R * w).sum(axis=1)` | broadcasting | 十倍+ |
| **`R @ w`** | **BLAS 优化** | **数十倍+** |
| `np.einsum` | 灵活但稍慢于@ | 接近@ |

**规模越大，差距越夸张**。N=500 或 5000 时，for 循环基本不可用。

## 🐼 Pandas 底层就是 NumPy

```python
df_slice.values       # → np.ndarray，同一块内存
df_slice.to_numpy()   # → np.ndarray，同上
df_slice @ weights    # 底层调的还是 NumPy 的 @，多了一步索引对齐
```

Pandas 的 `@` 比 NumPy 的 `@` 稍慢，因为要处理索引对齐。纯数值计算时，先 `.to_numpy()` 再算更快。

## ⚠️ 典型误用

1. **想算组合收益却用了 `*`**：`w * r` 给的是明细数组，不是总和标量
2. **`R @ R` 不报错但含义全错**：(T,N)@(N,T) 不是组合收益，是协方差结构
3. **`R * R` 不是矩阵乘法**：逐元素平方，是方差计算的中间步骤

## 📊 分散化效应的数学证明

50只等权组合的年化波动率 < 单只股票平均年化波动率，这就是**分散化投资(Diversification)** 的数学证据：

```
组合波动 ≈ 单股平均波动 × √(1/50 + 49/50 × 平均相关系数)
```

当相关系数 < 1 时，组合波动一定小于单股平均波动。

## 🧠 记忆口诀

```
* 是"展开"（broadcasting）— 维度不变或增大
@ 是"压缩"（contraction）— 内维消融，维度减小
```
