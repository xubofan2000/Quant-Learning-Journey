# Phase 1 - Week 2 - Day 1: 时序索引与缺失值填充总结

## 1. DatetimeIndex：解锁 Pandas 时间序列的灵魂钥匙

你在代码中执行的这两行看似平淡，实则是整个量化数据预处理的绝对基石：

```python
df['trade_date'] = pd.to_datetime(df['trade_date'])
df.set_index('trade_date', inplace=True)
```

* **为什么不能跳过？** 如果 `trade_date` 只是一个普通的字符串列，Pandas 会把它当成文本看待。你没法执行任何基于"日期范围"的切片操作（如 `df['2020-01':'2020-06']`），也没法用 `resample`、`rolling` 这些 Week 1 Day 7 你已经用过的时序神器。
* **`set_index` 的本质**：把 `trade_date` 从普通数据列"升格"为 DataFrame 的坐标轴。此后 Pandas 会在底层自动构建一棵基于日期的 B-tree 索引树，使得按日期范围的查询速度从 O(N) 线性扫描飙升到 O(log N) 二分查找。

## 2. 破坏性测试 (Chaos Engineering)

你用 `np.random.choice` 随机挑出 10 个日期，强行把收盘价替换为 `NaN`。这个操作在软件工程里有一个专业术语叫做**"混沌工程 (Chaos Engineering)"**——主动向系统注入故障来验证系统的容灾能力。

* **量化中的现实场景**：一只 A 股停牌 3 个月、API 因为网络抖动没抓到某天的数据、交易所当天本来就没开盘（节假日）……这些在真实数据管道中是家常便饭。你今天学会的这套"先破坏再修复"的方法论，确保了你的 Pipeline 对脏数据有感知和抵抗能力。

## 3. 前向填充 ffill()：最符合金融直觉的修复策略

```python
df['close_price_clean'] = df['close_price'].ffill()
```

当某天收盘价缺失时，`ffill()` 会**向上找到最近的一个有效值**，把它"拖"下来填充空缺位。

* **为什么不用均值填充？** 如果一只股票停牌前收盘价是 3.12 元，停牌 5 天后复牌涨到 3.50 元。用均值 `fillna(df.mean())` 填进去的是过去 5 年的平均价，可能是 4.00 元——在停牌期间凭空制造了一个从未发生过的价格跳跃，所有基于此计算的收益率和技术指标全部失真。
* **为什么不用后向填充 bfill？** `bfill()` 用的是"未来"的数据。在量化回测中，使用未来信息是最致命的禁忌（称为**前视偏差 Look-Ahead Bias**）。如果一只股票停牌后暴涨，用 bfill 会让停牌期间提前"知道"了涨后的价格，回测结果严重失真、策略收益虚高。
* **ffill 的金融语义**：如果今天没有新的交易价格产生，那么这只股票的"合理估值"就定格在最后一次成交价上。这是全球交易所的通用共识。

## 4. 验尸报告：你的运行结果解读

```
trade_date  close_price  close_price_clean
2019-06-17        3.120              3.120   ← 最后的有效价格
2019-06-18          NaN              3.120   ← 停牌！ffill 把 3.120 拖下来
2019-06-19        3.182              3.182   ← 恢复交易，用真实值
2019-06-20          NaN              3.182   ← 又停牌！ffill 拖 3.182
2019-06-21        3.297              3.297   ← 恢复
```

完美！每个 NaN 都被精确地替换成了「最近一个真实交易日的收盘价」，没有引入任何未来信息，也没有引入任何统计假象。
