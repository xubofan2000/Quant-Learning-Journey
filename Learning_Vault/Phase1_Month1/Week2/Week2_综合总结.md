## 🗺️ 知识地图

### 1. 时间序列的降维打击 (Day 1 - Day 2)
量化数据的本质就是时间序列。
- **时序对齐**：使用 `pd.to_datetime` 和 `set_index` 将普通 DataFrame 转换为真正具有时间灵魂的 `DatetimeIndex` 对象。只有有了时间索引，数据的对齐才不会错位。
- **缺失值处理**：停牌或节假日带来的数据缺失不能简单 drop。对于价格数据，最合理的假设是**价格不变**，使用 `.ffill()` (Forward Fill) 向前填充以避免未来函数。
- **滑动与位移**：
  - `.shift(1)`：获取昨日数据，这是计算今天收益率 `(P_t / P_{t-1}) - 1` 的核心动作。
  - `.rolling(window=20)`：滑动窗口，用于计算均线 (MA) 和历史波动率 (`.std()`)。

### 2. 金融特征工程的核心定律 (Day 3 - Day 4)
收益率不是简单的加减乘除，它有不同的物理意义。
- **单利收益率 (Simple Returns)** $\frac{P_t}{P_{t-1}} - 1$：适合截面加权（资产组合的加权和），跨期计算时需要用**连乘** `.cumprod()` 才能得到净值 (NAV)。
- **对数收益率 (Log Returns)** $\ln(\frac{P_t}{P_{t-1}})$：具有**时间可加性**。跨期收益可以用连续相加 `.cumsum()` 代替连乘，大幅降低了计算复杂度，是高频和理论推导的最爱。
- **超额收益 (Excess Return)**：通过 `pd.merge` 将个股数据与大盘指数对齐，计算个股脱离系统性风险后的独立超额表现（Alpha），这是多表关联 (`left join`) 的经典场景。

### 3. 数据存储的升维 (Day 5)
量化数据动辄 PB 级，CSV 已经无法满足工业化需求。
- **Parquet 列式存储**：与 CSV 行式文本不同，Parquet 是强类型的列式二进制格式。
- **三大优势**：巨大的压缩比，超高读取速度（尤其当你只需要分析特定的几列时），以及自动保存 Pandas 的数据类型（例如 `Datetime` 索引）。告别了每次读 CSV 还要 parse_dates 的痛苦。

### 4. 从脚本走向工程：OOP 与面板数据 (Day 6 - Day 7)
脚本 (Script) 只能完成单次任务，工程 (Engineering) 能够流水线作业。
- **面向对象重构 (OOP)**：使用类 (`class`) 封装状态（配置、数据字典）和方法（下载、清洗、指标计算、存储）。`DataPipeline` 类让复杂的处理流程一目了然。
- **面板数据 (Panel Data)**：扩展到 50 只股票的十年历史数据集。将多只股票拼接（Concat）并建立 MultiIndex，形成截面关联又顺延时间的三维结构。
- **压力测试**：验证千万级数据点在 Pandas 中的内存占用与计算耗时（往往发现瓶颈不在计算，而往往在循环）。

## 🛠️ 关键代码思维速查表

| 功能操作 | 典型代码片段 | 底层逻辑与适用场景 |
| :--- | :--- | :--- |
| **时序索引与填充** | `df.set_index('date').ffill()` | 停牌假定价格不变，避免未来数据泄露 |
| **时间位移(次日)** | `df['close'].shift(-1)` | 用于构建机器学习的预测目标 (`Target_Return_next`) |
| **对数收益率** | `np.log(df['close'] / df['close'].shift(1))` | 基于对数的差分，具有时间累加性 |
| **滑动标准差** | `df['returns'].rolling(20).std()` | 评估资产在过去 20 天的真实价格跳跃剧烈程度 |
| **列式存储写与读** | `df.to_parquet('A.parquet')` | 极速读写，自动保留所有数据类型格式 |
