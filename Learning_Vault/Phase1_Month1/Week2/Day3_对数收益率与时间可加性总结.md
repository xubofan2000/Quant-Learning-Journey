# Phase 1 - Week 2 - Day 3: 对数收益率与时间可加性总结

## 1. 两种收益率的本质区别

| | 简单收益率 (Simple Return) | 对数收益率 (Log Return) |
|---|---|---|
| **公式** | `r = (P_t - P_{t-1}) / P_{t-1}` | `r = ln(P_t / P_{t-1})` |
| **Pandas 写法** | `df['close_price'].pct_change()` | `np.log(df['close_price'] / df['close_price'].shift(1))` |
| **累计收益** | 必须**连乘** `(1+r₁)×(1+r₂)×...` | 直接**求和** `r₁+r₂+...` 再 `exp()` 还原 |
| **日度级别差异** | 几乎相同（万分之几） | 几乎相同 |
| **量化圈偏好** | 直觉易懂，给客户看报告用 | 建模首选，数学性质更优 |

## 2. 对数收益率的杀手级优势：时间可加性

这是今天最重要的一个知识点。假设你要算 2024 年全年累计涨了多少：

**简单收益率**需要 `prod()` 连乘——在数学上，乘法链条中任何一个数如果有微小的浮点误差，都会被后续的乘法不断放大（误差传播）。

**对数收益率**只需要 `sum()` 求和——加法的误差不会被放大，而且当你要对 50 只股票、252 个交易日的面板数据做批量累计时，加法操作可以充分利用向量化加速，比逐元素连乘快得多。

```python
# 连乘（慢，数值不稳定）
cumulative_simple = (1 + df['simple_return']).prod() - 1

# 求和（快，数值稳定）
cumulative_log = np.exp(df['log_return'].sum()) - 1
```

## 3. 踩坑记录：YTD 收益率的起点陷阱

今天遇到了一个经典的 off-by-one 边界 Bug：

**错误做法**：用 2024 年第一个交易日的收盘价做分母
```python
# ❌ 分母是 2024.1.2 的收盘价，漏掉了当天的涨跌
ground_truth = P_2024末 / P_2024首日 - 1
```

**正确做法（金融业务标准）**：用 2023 年最后一个交易日的收盘价做分母
```python
# ✅ 分母是 2023.12.29 的收盘价，完整计入 2024 年每一天的收益
last_price_2023 = df.loc[:'2023-12-31', 'close_price'].iloc[-1]
ground_truth = P_2024末 / last_price_2023 - 1
```

**为什么？** 基金行业算 YTD（Year-To-Date）收益，假设你在上一年最后一个交易日收盘时就已经持仓。2024.1.2 的涨跌本身就属于"2024 年收益"的一部分。如果用首日收盘价做分母，等于假设你在开年第一天收盘时才买入，完美踏空了开年的大涨/大跌——这个 1~2% 的误差在基金年报里是会被审计追责的。

**方法论升级**：在计算任何区间收益时，永远确保**收益率序列的起点**和**价格基准的起点**是同一天。这就是"殊途同归"原则——不管你用连乘、求和还是首尾直接算，三条路径的结果必须精确一致，否则一定有边界 Bug。

## 4. 对数收益率的数学直觉

```
ln(a/b) = ln(a) - ln(b)
```

这个恒等式意味着：**取对数后，除法变成了减法，乘法变成了加法**。这就是为什么累计收益从"连乘"退化成了"求和"——对数把乘法空间映射到了加法空间。在加法空间里，一切聚合操作（求和、求均值、做回归）都变得极其自然与高效。
