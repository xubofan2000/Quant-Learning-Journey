# 总结笔记模板与规范

> 供 agent 在生成 DayN_xxx总结.md 时参考。模板不是硬性框架，根据当天内容灵活增减章节。

## 完整模板

```markdown
# Week X Day Y: <中文标题> (<核心公式或技术关键词>)

## 1. 核心认知与代码解释
<今日最核心的概念解释。不超过 150 字。>
<最关键的代码片段（不超过 15 行），附中文注释解释"为什么这么做">

```python
# 关键代码示例
code_here
```

## 2. <技术专题标题>（灵活命名，例如：性能对比、API速查、数学推导）

<这一节根据当天内容动态命名和填充。常见类型：>
- 性能对比表（benchmark结果）
- API速查表（关键参数说明）
- 数学推导过程（公式+说明）
- 注意事项/陷阱

## 3. 整体学习进度与知识关联

*   **复盘前序 (Day 1 → Day N-1)：**
    *   <具体引用之前某天学到的内容，说明今天是它的延伸>
    *   例："Day 3 徒手推导的 Σ = (R_c.T @ R_c) / (T-1) 在今天成为了输入"
*   **当前里程碑 (Day N)：**
    *   <今天打通了什么，用一句话说清楚认知跃迁>
*   **预告引申 (下一步)：**
    *   <下一天/下一周会用到今天的什么内容>

## ⚠️ 薄弱点记录（仅在掌握度评估为"低"时生成）

*   **薄弱点 1**: <具体知识点>
    *   建议复习: <资料/方法>
*   **薄弱点 2**: ...
```

---

## 质量标准（逐条检查）

- [ ] 文件名格式正确：`DayN_<中文描述>总结.md`
- [ ] 标题行包含当天核心技术关键词（如数学符号、工具名）
- [ ] 核心认知章节不超过 150 字（精炼，不是复制代码注释）
- [ ] 代码片段选最核心的逻辑，不超过 15 行/块
- [ ] 数学公式用 LaTeX：行内 `$formula$`，独立行 `$$formula$$`
- [ ] 知识关联章节**具体引用**前序 Day（"Day 3 的 Σ"，而非"之前学过的内容"）
- [ ] 如掌握度低，生成「薄弱点记录」章节

## 闪卡/习题生成规范

在 Workflow A Step 5 中生成，格式如下：

```markdown
## 📚 今日学习检验 - Week X Day Y

### 闪卡（概念验证）

**Q1**: <概念性问题>
> 提示：<关键词提示>
<details><summary>答案</summary>
<答案内容>
</details>

**Q2**: ...（共 3-5 道）

---

### 代码习题

**题目 1**：<描述一个需要用今日知识解决的小任务>

```python
# 填写缺失代码
import numpy as np
w = np.array([0.3, 0.3, 0.4])
Sigma = ___  # 3x3 协方差矩阵

# 目标：用一行代码计算组合方差
portfolio_var = ___
```

**预期输出**：`portfolio_var ≈ 0.00123`

<details><summary>参考答案</summary>

```python
portfolio_var = w.T @ Sigma @ w
```
</details>
```

### 闪卡出题原则

- 每道题聚焦**一个**核心概念，不出综合题
- 数学类：给公式 → 问代码实现，或给代码 → 问数学含义
- API 类：给场景 → 问用哪个函数/参数
- 陷阱类：给错误代码 → 问问题在哪
- 代码习题：填空/改错，不要让用户从零写（减少负担）
